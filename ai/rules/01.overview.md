# Project Overview: PKCard TCG

## 1. Core Identity
PKCard is a local 2-player, browser-based Trading Card Game (TCG) prototype built with React and Vite. It emphasizes fluid drag-and-drop mechanics, synthesized audio, and a premium visual experience.

## 2. Architecture & State Management
- **Framework**: React + Vite.
- **State Handling**: A single centralized `gameState` object managed in `GameArena.jsx`. All state updates MUST use `structuredClone` for strict immutability.
- **Routing**: React Router v6 in `App.jsx`. Navigation bar is hidden on `/`, `/setup`, and `/battle`.
- **Data Layer**: `src/models/cards.js` serves as the local database for all cards (Pokemon, Energy, Trainers) and theme deck generation logic.
- **Global Stores** (Zustand + persist): `useAuthStore` (auth/user), `useCardStore` (custom decks), `useThemeStore` (dark/light theme, persisted to localStorage).

## 3. Page Architecture

| Route | Component | Has Nav | Theme-aware |
|---|---|---|---|
| `/` | `HomePage` | ❌ | ❌ hardcoded dark |
| `/setup` | `SetupPage` | ❌ | ❌ hardcoded dark |
| `/battle` | `BattlePage` → `GameArena` | ❌ | ⚠️ partial (CSS classes used, but container hardcoded) |
| `/login` | `LoginPage` | ✅ | ✅ inherits body |
| `/studio` | `DeckListPage` | ✅ | ✅ CSS tokens |
| `/studio/new` | `StudioPage` | ✅ | ✅ CSS tokens |
| `/studio/edit/:deckId` | `StudioPage` | ✅ | ✅ CSS tokens |
| `/profile` | `ProfilePage` | ✅ | ✅ inherits body |

**Theme system:** `body { background: var(--theme-bg-global) }` is toggled via `body.dataset.theme`
(`"dark"` = default, `"light"` = Theme B Light Holographic Glass). Token definitions live in `src/index.css`.

**⚠️ Known gap:** `HomePage` and `SetupPage` paint their own `background: '#060918'` inline,
bypassing the CSS token system entirely. They are intentionally self-contained dark-cosmic pages.
`Lobby.jsx` in `src/components/` is legacy code — it is no longer routed and can be archived.

## 3. Gameplay Mechanics
- **Card Types**: 
  - Basic Pokemon (deploy to active/bench)
  - Stage 1 Evolution (drop onto matching basic Pokemon; inherits damage/energy)
  - Energy (attach to Pokemon to meet attack costs, 1 per turn)
  - Trainer Cards (drop on board/Pokemon for immediate effects)
- **Turn Flow**: Draw phase on turn start -> Attach Energy (optional) -> Play Cards -> Attack (ends turn).
- **Win Conditions**: 
  1. Take all Prize Cards by knocking out opponent Pokemon.
  2. Opponent has no Active Pokemon and no Bench Pokemon to promote (Bench Out).
  3. Opponent cannot draw a card at the start of their turn (Deck Out).
