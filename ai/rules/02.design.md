# Design & Aesthetics Guidelines

## 1. Core Visual Principles
- **Modern & Premium**: The UI must look like a high-end, polished game, not a basic web app.
- **Glassmorphism**: Use `backdrop-filter: blur()`, semi-transparent backgrounds, and glowing borders (`glass-panel`).
- **Dark Theme Priority**: Rely on deep cosmic colors (`#0f172a`, `#1e1b4b`) to make card colors and particle effects pop.

## 2. Card Styling Rules
- **Basic Pokemon**: Standard cosmic/slate gradient (`#334155` to `#0f172a`).
- **Stage 1 Evolution**: Premium holographic foil aesthetic. Use rich iridescent gradients (e.g., Indigo to Purple to Gold) with an inner gold glow/border. MUST include the "進化 (Evolution)" badge.
- **Stage 2 Evolution**: Ultimate prismatic/diamond foil aesthetic. Use hyper-luxurious gradients (e.g., Platinum/Silver to Diamond Blue to Gold, or full rainbow holographic), with an intense pulsing aura or animated glow to signify maximum power. MUST include the "二階進化 (Stage 2)" badge.
- **Trainer/Item Cards**: Tech/Utility aesthetic. Use Teal/Cyan gradients (`#14b8a6` to `#0f766e`).
- **Energy Cards**: Deep vibrant gradients matching their element type (Fire=Red, Water=Blue, Grass=Green, Electric=Yellow).

## 3. Interaction & Animation
- **Physical Feel**: Card interactions MUST feel physical. Utilize the native HTML5 Drag and Drop API.
- **Micro-animations**: Elements should scale up slightly (`scale(1.05)`) and increase their `box-shadow` intensity on hover.
- **Transitions**: State changes (playing a card, changing turns) should have smooth CSS fade-ins (`animate-fade-in`) or slide effects.

## 4. Audio Design (Web Audio API)
- **No External Assets**: DO NOT use external `.mp3` or `.wav` files. All audio must be synthesized dynamically using the Web Audio API.
- **8-bit Retro Style**: BGM and SFX (attacks, placing cards, errors, victory) should utilize oscillators (square, sawtooth) to evoke a nostalgic, retro 8-bit gaming feel.
