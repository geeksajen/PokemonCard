# 卡牌工坊 (Deck Studio) UI/UX 重構規格書

## 1. 概述 (Overview)
為了提供更清晰、直覺的牌組管理體驗，卡牌工坊將從目前的「單一編輯頁面」拆分為「列表頁」與「編輯頁」的雙層次結構。這樣的設計更貼近主流卡牌遊戲的邏輯。

---

## 2. 頁面結構 (Pages)

### 2.1 牌組列表頁 (Deck List Page)
**建議路由:** `/studio`
**功能與介面設計:**
- **頭部導覽:** 顯示「卡牌工坊 / 我的牌組」。
- **新增按鈕:** 一個明顯的「➕ 建立新牌組」的大型按鈕或卡片區塊。
- **牌組展示區 (Grid / List):** 列出玩家目前儲存在 LocalStorage 的所有牌組，每副牌組呈現以下資訊：
  - **封面圖片:** 顯示玩家設定的封面卡牌 (若無則顯示預設卡背)。
  - **牌組名稱:** 玩家自訂的名稱。
  - **卡牌張數:** 提示是否滿 27 張。
  - **操作按鈕:** 「編輯」與「刪除」按鈕。

### 2.2 牌組編輯頁 (Deck Edit Page)
**建議路由:** `/studio/edit/:deckId` 或 `/studio/new`
**功能與介面設計:**
- **繼承現有設計:** 保留目前開發完成的雙欄佈局（左側卡牌庫、右側當前牌組清單）。
- **頂部工具列:** 增加「⬅ 返回列表」按鈕。
- **移除冗餘功能:** 先前加入的「📁 讀取牌組」Modal 視窗將被移除，由列表頁完全取代。
- **儲存按鈕狀態:**
  - 根據卡牌張數限制按鈕狀態 (未滿 27 張不可儲存)。

---

## 3. 使用者流程 (User Flow)

### 3.1 新增牌組流程 (Create Deck Flow)
1. **進入工坊:** 點擊首頁導覽列進入 `/studio` 牌組列表頁。
2. **建立新檔:** 點擊「新增牌組」，跳轉至編輯頁 (`/studio/new`)，此時右側牌組清單為空。
3. **組建卡牌:** 玩家手動挑選 27 張卡牌，或是點擊「✨ 智能組牌」。
4. **點擊儲存:** 玩家點選「儲存牌組」。
5. **命名提示:** 系統彈出對話框要求輸入「牌組名稱」。
6. **完成:** 存檔成功，系統自動分配 `deckId`，並可選擇留在編輯頁繼續調整，或是跳轉回列表頁。

### 3.2 編輯牌組流程 (Edit Deck Flow)
1. **進入工坊:** 在 `/studio` 列表頁看見已儲存的牌組。
2. **選擇編輯:** 點選其中一副牌組，跳轉至編輯頁 (`/studio/edit/:deckId`)。系統會自動將該牌組的卡牌載入右側清單。
3. **進行修改:** 玩家自由增減卡牌。
4. **點擊儲存:** 玩家點選「儲存牌組」。
5. **快速更新:** 系統**不會**再跳出重新命名的提示，而是直接將修改覆蓋寫入原有的 `deckId`。
6. **完成:** 顯示「儲存成功」的提示。
