# 遊戲大廳 (Main Lobby) UI/UX 升級企劃書

## 🔍 現況分析 (Current Status)
目前的 `Lobby.jsx` 架構更偏向於**「單機對戰設定區 (Local Match Setup)」**。
將「選擇 P1 牌組」、「選擇 P2 牌組」和「開始對戰」全部集中在進入遊戲的第一個畫面，這在功能開發期非常實用。但若以現代化數位卡牌遊戲（如《Pokémon TCG Live》、《爐石戰記》）的標準來看，缺乏作為遊戲中央樞紐 (Central Hub) 的沉浸感與導覽擴充性。

## 🎯 重構目標 (Redesign Goals)
將大廳升級為**「中央樞紐 (Main Hub)」**，把功能設定藏到第二層，將首頁第一層的視覺空間保留給「玩家成就/主視覺展示」與「系統主要入口」。

---

## 🛠️ UI/UX 具體藍圖 (Blueprint)

我們計畫將目前的單一首頁拆分為**「雙層架構」**：

### 第一層：Main Hub (真正的首頁大廳)
這是玩家一進入遊戲所看到的沉浸式大廳。

1. **環境氛圍 (Environment)**
   - 背景：深宇宙星空背景（繼承全域設計規範）。
   - 音效：播放 8-bit Retro Style 的大廳待機音樂。

2. **主視覺焦點 (Visual Showcase)**
   - 位置：畫面正中央偏右。
   - 內容：展示玩家設定的「王牌寶可夢」或「當前選定牌組的封面卡」的巨大去背立繪（可加上緩慢的漂浮動畫與毛玻璃底座），建立玩家的情感投射與帶入感。

3. **資訊與導覽列 (Navigation & Status)**
   - **右上/左上角 (玩家狀態)**：玩家頭像、名稱、等級/階級，以及貨幣資產。
   - **左下/側邊欄 (系統入口)**：
     - 🛠️ 卡牌工坊 (Collection / Studio)
     - 🎁 商店/抽卡 (Shop - 預留擴充)
     - 📜 任務/活動 (Quests - 預留擴充)
   - **右下角 (行動呼籲 CTA)**：一個最顯眼、帶有脈衝發光動畫的巨大 **「PLAY (開始對戰)」** 按鈕。

### 第二層：Match Setup (對戰前夕設定)
當玩家點擊首頁的「PLAY」按鈕後，才會過渡到此介面（即目前的 `Lobby.jsx` 邏輯）。

1. **版面配置 (Layout)**
   - 畫面中央顯示強烈的 `VS` 視覺特效。
   - **左側 (Player)**：顯示並確認玩家目前選擇的出戰牌組。
   - **右側 (Opponent)**：選擇對手模式（對戰電腦 AI 或是 雙人面連面模式），並選擇電腦的牌組難度/屬性。
2. **準備完畢 (Ready)**
   - 確認設定後，點擊「確認出戰」，畫面進入轉場動畫（Turn Transition Splash Screen），正式載入 `GameArena` 戰鬥場景。

## 🚀 開發階段建議 (Phased Implementation)
- **Phase 1**: 將現有的 `HomePage.jsx` 抽換為新的 Main Hub 視覺介面，實作「王牌寶可夢」主視覺與大顆的 PLAY 按鈕。
- **Phase 2**: 將現有的 `Lobby.jsx` 對戰設定邏輯移至點擊 PLAY 彈出的 Modal 或下一個路由頁面 (`/setup`)。
- **Phase 3**: 串接大廳的玩家狀態列（讀取 local storage 或 store 中的虛擬玩家資料）。
